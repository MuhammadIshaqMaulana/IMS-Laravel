Saya ingin aplikasinya secara bertahap kita buat aplikasi IMS ini memiliki fitur universal dan sesimple Sortly.

Jadi kalau saya mau mengikuti sortly dengan segala fitur simple dan dinamisnya, seperti membuat beragam item, dengan berbagai tag dan custom fieldnya masing2(boolean), sistem folder yang sama persis seperti kita mengatur file folder. Sepertinya custom field dan tag tidak terlalu beeda fungisnya. karena sepertinya tipenya boolean dan berfungsi untuk menambah atribut pada item/file.

Karena sepertinya refactor ini akan cukup lama, jadi saya ingin kamu agar sekarang fokus buatkan fase-fase untuk refactor ini, mulai dari perencanaan, perombakan database, logika, UI, hingga selesai. tidak masalah faseya banyak karena yang penting 1 fase tuh jangan terlalu padat, karena saya juga akan push ke git untuk setiap fase.
SKU otomatis.

Berikut lanjutan nomor 5 pada chat sebelumnya:
5. default fieldnya adalah:
- name
- quantity
- unit of measure
- min. level
- price
- tags 
- notes
- option 'this item has variant' (ini memunculkan input agar user bisa input varian)(setelah dibuat, setiap varian item akn menjadi item individu masing-masing)


6. kemampuan untuk memilih multiple item, baik memilih 1 per 1, semuanya, atau berdasarkan filter. Kemampuan ini digunakan untuk:
- bulk edit
	- name(add/replace, Add prefix/suffix, sequential number prefix/suffix)
	- min. level(stok minimum)(add/replace)
	- Price(add/replace, increment, decrement, multiply, divide)(karena price tidak boleh desimal, jadi bulatkan saja keatas/kebawah dan berikan keterangakan di UI itu dibulatkan keatas atau kebawah)
	- notes(add/replace, Add prefix/suffix, sequential number prefix/suffix)
	- tags(add/remove)
	- custom fields
	
* Walaupun diberi kebebasan untuk mengedit, tapi sortly tetap ketat dalam membatasi input. Seperti mencegah user mengalikan harga sampai out of max input.
